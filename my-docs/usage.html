Note

class pymusic.note.Note(pitch: Pitch, note_length: NoteLength, duration: int, measure: Measure, instrument: str, voice: int, display_info: dict)

Represents a single musical note

Internal representation of the note element within Music XML. This representation is focused on musically identifiable / interesting aspects, to aid in comparisons and analysis.

Parameters

  • pitch (Pitch) – the pitch_soup value of this note

  • note_length (NoteLength) – the notated length

  • duration (int) – How long this note sounds for

  • measure (str) – the measure (bar) that contains this note

  • instrument (str) – the instrument which plays this note

  • voice (str) – the voice which plays this note

  • display_info (dict) – all information relating to the display of this note on the score

Note Length

class pymusic.note_length.NoteLength(original_type)

Represents the length of the note.

Internal representation of the note-type-value type in MusicXML. The British note names are used to refer to the notes duration through this program as I am Australian, and to help differentiate between the duration as written, and the duration when performed - as this may vary due to musical style and preference.

Parameters

  • length_name (str) – length name

  • amount (int) – the amount of time

Note Name

class pymusic.note_name.NoteName(value)

The names of each note in an octave as defined by ASPN.

Each octave starts at C, so the value assigned by the enum reflects this. Internal implementation of the step data type defined in Music XML.

Octave

class pymusic.octave.Octave(value)

Contains a value representing the octave of interest.

There are 9 octaves with 1 corresponding to the lowest octave and 8 to the highest octave. Middle C is found in octave 4. See https://www.w3.org/2021/06/musicxml40/musicxml-reference/elements/octave/ for more information regarding octaves in music XML.

pymusic.pitch.Pitch.create_from_xml_soup(pitch_soup: Tag) → Pitch

Creates a Pitch instance from the provided MusicXML.

The provided XML must contain both an <octave> and <step> element which must contain the octave number and string note name respectively. The following example creates a Pitch corresponding to middle C.

<pitch_soup>
    <octave>4</octave>
    <step>C</step>
</pitch_soup>
Parameters

pitch_soup (bs4.Tag) – the XML containing this pitch_soup and passed through BeautifulSoup.

Returns

the created pitch_soup.

Return type

Pitch

Raises

ValueError – If the XML is cannot be parsed correctly to form a pitch_soup.
